# Portfolio System

Version: v1.0.5-basic

## 目的

Portfolio System 用來管理 Louis 目前持股、分類、健康度與操作狀態。

它回答：

> 我目前持有什麼？每一檔角色是什麼？是否需要調整？

## 持股分類

### 核心

長期持有，不因短線 KD 過熱單獨賣出。

例：
- 0050
- 台積電
- Microsoft
- NVIDIA

### 波段

基本面可接受，但可依技術面、RS、OBV、MACD、主力成本調整。

例：
- 南電
- 欣興
- 景碩
- 漢唐
- 聯發科
- 國巨
- 鴻海

### 觀察

尚未完全符合 LIOS 條件，或技術面偏弱。

例：
- 華旭先進
- 彩晶

### 成長

高波動、高成長、小部位。

例：
- SPCX

## 健康度

健康度以 0~100 顯示。

建議解讀：

- 90~100：高信心
- 80~89：偏多
- 70~79：中性偏觀察
- 60~69：偏弱
- 60 以下：不適合加碼，需評估退出或換股

## 信心等級

- ⭐⭐⭐⭐⭐：高信心，可逢回布局
- ⭐⭐⭐⭐☆：偏多，持有等待機會
- ⭐⭐⭐☆☆：中性，觀察
- ⭐⭐☆☆☆：偏弱，控制部位
- ⭐☆☆☆☆：轉弱，評估退出

## 固定輸出欄位

Portfolio Report 需包含：

- 股票
- 分類
- 股數
- 成本
- 現價
- 損益
- 健康度
- 信心等級
- 動作建議
- 備註

## 更新頻率

### 每日

更新：
- 現價
- 損益
- 動作建議
- 燈號

### 每週

更新：
- 健康度
- 分類是否合理
- 弱勢持股清單

### 每月

更新：
- 是否需要換股
- 是否需要調整核心 / 波段 / 觀察分類
- 是否有重複持股或過度集中

## 原則

- 不因成本價決定買賣
- 弱勢股不攤平
- 核心股不因短線 KD 過熱單獨賣出
- 波段股要尊重 Decision Engine
- 觀察股若長期未改善，需列入 Review
