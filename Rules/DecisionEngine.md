# Decision Engine

Version: v1.0.4-basic

## 用途

Decision Engine 用來回答：

- 可以買嗎？
- 可以加碼嗎？
- 要續抱嗎？
- 要減碼嗎？
- 要停損嗎？

## 評分順序

1. 基本面
2. 估值
3. 趨勢 / RS
4. OBV
5. MACD
6. KD
7. 法人 / 籌碼
8. 成交密集區 / 主力成本
9. 風險室燈號
10. 反方檢查

## 建議分數配置

| 項目 | 分數 |
|---|---:|
| 基本面 | 25 |
| 估值 | 15 |
| 趨勢 / RS | 15 |
| OBV | 10 |
| MACD | 10 |
| KD | 5 |
| 法人 / 籌碼 | 10 |
| 成交密集區 / 主力成本 | 5 |
| 風險室 / 資產配置 | 5 |
| 合計 | 100 |

## 決策燈號

- 90~100：可以分批布局
- 80~89：觀察，等待更好位置
- 70~79：持有，不新增部位
- 60~69：不建議加碼
- <60：避免或評估退出

## Kill Switch

以下情況直接不能買：

- 基本面惡化
- 財報爆雷
- 公司治理問題
- 趨勢完全走空
- OBV / RS 持續惡化
- 風險室為紅燈

## 買入流程

買入前必須回答：

1. 基本面是否值得持有？
2. 估值是否合理？
3. RS 是否至少中性以上？
4. OBV 是否沒有明顯惡化？
5. MACD 是否改善或即將改善？
6. KD 是否不是極端過熱？
7. 法人籌碼是否不是明顯反向？
8. 是否接近主力成本區或突破後有效整理？
9. 風險室是否允許新增部位？

## 加碼流程

加碼比買入更嚴格。

加碼需至少符合三項：

- RS 轉強
- OBV 上升
- MACD 綠柱縮短或翻紅
- KD 黃金交叉且未過熱
- 回測主力成本區守住
- 法人回補
- 基本面或月營收改善

## 續抱流程

續抱條件：

- 基本面未變
- 趨勢未破壞
- OBV 未明顯出貨
- MACD 未進入明顯空方擴散
- 跌破支撐後有修復跡象

核心股不因 KD 過熱單獨賣出。

## 減碼 / 停損流程

減碼條件：

- RS 持續轉弱
- OBV 持續下降
- MACD 綠柱擴大
- 跌破主力成本區後無法收回
- 法人連續賣超
- 基本面不如預期
- 原始買進理由消失

## 反方檢查

每次得到「可以買」的結論後，必須再問：

> 如果這次判斷錯了，最可能錯在哪裡？

常見反方：

- 估值過高
- 資金輪動結束
- 法人短線出貨
- 大盤風險升高
- 題材已過度反映
- 指標只是短線反彈，不是趨勢反轉

## 固定輸出格式

每次 Decision Engine 輸出需包含：

1. 決策問題：買 / 加碼 / 續抱 / 減碼 / 停損
2. 總分
3. 燈號
4. 支持理由
5. 反方風險
6. 建議動作
7. 是否需要記錄到 Journal
